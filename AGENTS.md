# Agents Guide

## Project Overview

Open benchmark for evaluating speech-to-text providers across 5 locales (en-US, es-MX, tr-TR, vi-VN, zh-CN) with 4,270 annotated utterances. Audio is 24kHz mono from real customer service phone conversations.

## Repository Layout

```
manifest.json                  # Source of truth — all utterance IDs, ground truth transcripts, audio paths
scripts/
  download_audio.py            # Downloads audio from HuggingFace (requires HF_TOKEN)
  transcribe.py                # Runs provider APIs to generate transcripts + latency.json
  compare_transcripts.py       # Compare transcripts between two submission bases
  latency_stats.py             # Compute p50/p95 latency from latency.json files
  requirements.txt             # Python deps for scripts
scoring/
  validate.py                  # Submission format validation (open source, safe to run locally)
  normalize.py                 # LLM-based transcript normalization (requires scoring/prompts.py)
  score.py                     # Metrics computation (requires scoring/prompts.py)
  metrics.py                   # Core metric implementations (WER, quality, significant WER)
  llm.py                       # OpenAI API wrapper for scoring
  prompts.py                   # ⚠️ GITIGNORED — injected from GitHub secret in CI
  requirements.txt             # Python deps for scoring
submissions/
  raw/<provider>/              # Raw transcripts + latency.json + metadata.yaml
  normalized/<provider>/       # LLM-normalized transcripts (auto-generated by scoring pipeline)
results/
  <provider>/scores.json       # Per-provider scoring results (includes latency p50/p95)
  leaderboard.json             # Aggregated leaderboard data
leaderboard/                   # React/Vite web app for the leaderboard
latency_runs/                  # Re-run transcripts with latency capture (not committed)
audio/                         # ⚠️ GITIGNORED — downloaded from HuggingFace
run_all.sh                     # Maintainer script to normalize + score + latency stats
.env                           # ⚠️ GITIGNORED — API keys for providers and OpenAI
```

## Key Concepts

- **`manifest.json` is the source of truth.** It defines which utterances exist, their ground truth transcripts, and audio paths. All scripts (transcribe, validate, normalize, score) read from it.
- **`scoring/prompts.py` is secret.** It is stored as a GitHub secret (`SCORING_PROMPTS_PY`) and injected at CI time. It is not available locally for external contributors. Normalization and scoring cannot be run locally without it.

## Common Workflows

### Download audio from HuggingFace
```bash
HF_TOKEN=... .venv/bin/python scripts/download_audio.py
```
Exports .wav files to `audio/<locale>/`. The HuggingFace dataset (`sierra-research/mu-bench`) is public but gated — users must request access.

### Transcribe with a provider
```bash
.venv/bin/python scripts/transcribe.py \
  --provider deepgram-nova3 \
  --output-dir submissions/raw/deepgram-nova3 \
  --concurrency 10
```
Providers: `deepgram-nova3`, `google-chirp3`, `azure`, `elevenlabs-scribe-v2`, `openai-gpt4o-transcribe`. The script reads `manifest.json` to build the work list and skips utterances that already have output files. Per-utterance latency is recorded to `latency.json` in the output directory.

### Compare transcripts between runs
```bash
.venv/bin/python scripts/compare_transcripts.py \
  --old-base submissions/raw \
  --new-base latency_runs \
  --manifest manifest.json
```
Reports exact-match rates per provider and overall.

### Compute latency statistics
```bash
# Single provider
.venv/bin/python scripts/latency_stats.py \
  --submission-dir submissions/raw/<provider> \
  --manifest manifest.json \
  --output-dir results/<provider>

# All providers
.venv/bin/python scripts/latency_stats.py \
  --base-dir submissions/raw \
  --manifest manifest.json \
  --results-dir results
```
Reads `latency.json`, computes p50/p95 per locale and overall, merges into `scores.json`.

### Validate a submission
```bash
.venv/bin/python scoring/validate.py submissions/raw/<provider> --manifest manifest.json
```
Safe to run locally. Checks metadata.yaml, file names against manifest, file sizes, encoding. Optionally validates `latency.json` if present.

### Normalize + score (maintainers only)
```bash
source .env
.venv/bin/python -m scoring.normalize --submission-dir submissions/raw/<provider> --manifest manifest.json
.venv/bin/python -m scoring.score --submission-dir submissions/raw/<provider> --manifest manifest.json
.venv/bin/python scripts/latency_stats.py --submission-dir submissions/raw/<provider> --manifest manifest.json --output-dir results/<provider>
.venv/bin/python -m scoring.update_leaderboard
```
Requires `scoring/prompts.py` (from GitHub secret) and `OPENAI_API_KEY`. In CI, triggered by a maintainer commenting `/score` on a PR.

### Run all providers (maintainers only)
```bash
./run_all.sh
```

## CI Workflows

| Workflow | Trigger | What it does |
|----------|---------|-------------|
| `validate-submission.yml` | PR touching `submissions/raw/` | Validates submission format |
| `score-submission.yml` | `/score` comment on PR | Normalizes, scores, posts results |
| `post-merge-score.yml` | PR merged to main | Re-scores and commits results to main |
| `deploy-leaderboard.yml` | Push to main touching `leaderboard/` or `results/` | Builds and deploys leaderboard to GitHub Pages |

## Submission Format

Each submission is a directory under `submissions/raw/<provider>/` containing:
- `metadata.yaml` with fields: `model`, `organization`, `version`, `date` (required), `contact`, `notes` (optional)
- `<locale>/<utterance-id>.txt` files — one per utterance, plain text transcript only
- `latency.json` (optional) — maps utterance IDs to API response time in milliseconds

File names must match utterance IDs in `manifest.json`. Missing utterances are scored as full errors. Extra files are ignored with a warning.
