# Agents Guide

## Project Overview

Open benchmark for evaluating speech-to-text providers across 5 locales (en-US, es-MX, tr-TR, vi-VN, zh-CN) with 4,270 annotated utterances. Audio is 8kHz mono from real customer service phone conversations.

Leaderboard: [research.sierra.ai/mubench](https://research.sierra.ai/mubench)

## Repository Layout

```
manifest.json                  # Source of truth — all utterance IDs, ground truth transcripts, audio paths
pyproject.toml                 # Python deps (base + [tools], [scoring], [transcribe] extras)
scripts/
  download_audio.py            # Downloads audio from HuggingFace (requires HF_TOKEN)
  transcribe.py                # Runs provider APIs to generate transcripts + latency.json
  latency_stats.py             # Compute p50/p95 latency from latency.json files
  compare_transcripts.py       # Diff transcripts between two submission bases
  significance_test.py         # Statistical significance between providers
  verify_data.py               # Verify downloaded audio against manifest
scoring/
  validate.py                  # Submission format validation (safe to run locally)
  normalize.py                 # LLM-based transcript normalization (requires scoring/prompts.py)
  score.py                     # Metrics computation (requires scoring/prompts.py)
  metrics.py                   # Core metric implementations (WER, significant WER)
  llm.py                       # OpenAI API wrapper for scoring
  update_leaderboard.py        # Regenerates results/leaderboard.json from all scores.json files
  prompts.py                   # GITIGNORED — injected from GitHub secret in CI
submissions/
  SUBMITTING.md                # Public-facing submitter guide
  raw/<provider>/              # Raw transcripts + latency.json + metadata.yaml
  normalized/<provider>/       # LLM-normalized transcripts (auto-generated by scoring pipeline)
results/
  <provider>/scores.json       # Per-provider scoring results (includes latency p50/p95)
  <provider>/details/          # Per-utterance scoring detail files
  leaderboard.json             # Aggregated leaderboard data
  significance.json            # Pairwise statistical significance
leaderboard/                   # React/Vite web app for the leaderboard
audio/                         # GITIGNORED — downloaded from HuggingFace
.env                           # GITIGNORED — API keys for providers and OpenAI
```

## Key Concepts

- **`manifest.json` is the source of truth.** It defines which utterances exist, their ground truth transcripts, and audio paths. All scripts read from it.
- **`scoring/prompts.py` is secret.** Stored as a GitHub secret (`SCORING_PROMPTS_PY`) and injected at CI time. Normalization and scoring cannot be run locally without it.
- **`submissions/normalized/` and `results/` are auto-generated.** The post-merge CI pipeline produces these and commits them to main. Do not edit them by hand.
- **`leaderboard/src/data/data.js` is auto-generated** from `results/leaderboard.json` by `leaderboard/scripts/inject-data.js`. Do not edit by hand.

## Common Workflows

### Download audio from HuggingFace
```bash
HF_TOKEN=... .venv/bin/python scripts/download_audio.py
```
Exports .wav files to `audio/<locale>/`. The HuggingFace dataset (`sierra-research/mu-bench`) is public but gated.

### Transcribe with a provider
```bash
.venv/bin/python scripts/transcribe.py \
  --provider deepgram-nova3 \
  --output-dir submissions/raw/deepgram-nova3 \
  --concurrency 10
```
Providers: `deepgram-nova3`, `google-chirp3`, `azure`, `elevenlabs-scribe-v2`, `openai-gpt4o-mini-transcribe`. The script reads `manifest.json` to build the work list and skips utterances that already have output files. Per-utterance latency is recorded to `latency.json`.

Note: for latency to be comparable across providers, measure at **concurrency = 1**. Higher concurrency inflates numbers due to queueing.

### Compute latency statistics
```bash
# Single provider
.venv/bin/python scripts/latency_stats.py \
  --submission-dir submissions/raw/<provider> \
  --manifest manifest.json \
  --output-dir results/<provider>

# All providers
.venv/bin/python scripts/latency_stats.py \
  --base-dir submissions/raw \
  --manifest manifest.json \
  --results-dir results
```
Reads `latency.json`, computes p50/p95 per locale and overall, merges into `scores.json`.

### Validate a submission
```bash
.venv/bin/python scoring/validate.py submissions/raw/<provider> --manifest manifest.json
```
Safe to run locally. Enforces required metadata fields, required `latency.json` with locale-prefixed keys, transcript coverage for each shipped locale, file sizes, encoding. Partial-locale submissions are allowed.

### Normalize + score (maintainers only)
```bash
source .env
.venv/bin/python -m scoring.normalize --submission-dir submissions/raw/<provider> --manifest manifest.json
.venv/bin/python -m scoring.score --submission-dir submissions/raw/<provider> --manifest manifest.json
.venv/bin/python scripts/latency_stats.py --submission-dir submissions/raw/<provider> --manifest manifest.json --output-dir results/<provider>
.venv/bin/python -m scoring.update_leaderboard
```
Requires `scoring/prompts.py` (from GitHub secret) and `OPENAI_API_KEY`. In CI, triggered by a maintainer commenting `/score` on a PR.

## CI Workflows

| Workflow | Trigger | What it does |
|----------|---------|-------------|
| `lint.yml` | PR touching `scripts/`, `scoring/`, or `leaderboard/` | Runs ruff (Python) and eslint (JS) |
| `validate-submission.yml` | PR touching `submissions/raw/` | Validates submission format, posts result as PR comment |
| `score-submission.yml` | `/score` comment on PR | Normalizes, scores, posts results as PR comment |
| `post-merge-score.yml` | PR merged to main touching `submissions/raw/` | Re-scores and commits results + normalized transcripts to main |
| `deploy-leaderboard.yml` | Push to main touching `leaderboard/` or `results/` | Builds and deploys leaderboard to GitHub Pages |

The full scoring pipeline on merge: `post-merge-score.yml` runs normalize → score → update_leaderboard → commit. That commit updates `results/`, which triggers `deploy-leaderboard.yml` to rebuild and deploy the site.

## GitHub Secrets

| Secret | Used by | Purpose |
|--------|---------|---------|
| `SCORING_PROMPTS_PY` | score-submission, post-merge-score | LLM prompt templates written to `scoring/prompts.py` |
| `OPENAI_API_KEY` | score-submission, post-merge-score | API key for GPT-4.1 (normalization + scoring) |

These are configured in the `scoring` environment on the repo.

## Submission Format

Each submission is a directory under `submissions/raw/<provider>/` containing:
- `metadata.yaml` — required fields: `model`, `organization`, `version`, `date`; optional: `contact`, `notes`
- `<locale>/<utterance-id>.txt` — plain text transcript, one per utterance
- `latency.json` — required, maps `"<locale>/<utterance_id>"` to per-request latency in ms (measured at concurrency=1)

File names must match utterance IDs in `manifest.json`. Within each included locale, every utterance must have a transcript and a matching latency entry. Partial-locale submissions are allowed. Full contract lives in `submissions/SUBMITTING.md`.
